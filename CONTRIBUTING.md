# 贡献指南

感谢你愿意改进 Exam Clock。无论是修复流程、补充预设、改进无障碍体验还是完善文档，都欢迎参与。

## 开始之前

项目是静态站点，不需要安装依赖。建议使用本地 HTTP 服务运行：

```bash
python3 -m http.server 8000
```

访问 <http://localhost:8000>，并在常用桌面与移动端宽度下检查页面。

## 提交建议

1. 从 `main` 创建一个目的明确的分支。
2. 保持改动聚焦：功能、样式和文档的无关改动请拆分提交。
3. 更新内置考试数据时，说明信息来源、适用年份和地区；不要把未经核验的安排写成官方规则。
4. 修改 `sw.js` 的预缓存资源或应用核心文件时，请更新 `CACHE_NAME`，并验证刷新后能取得新版资源。
5. 提交 Pull Request 前，至少完成以下人工检查：
   - 可开始、暂停、重置并切换一个内置预设；
   - 可创建、保存、编辑、应用和删除一个自定义预设；
   - 收藏/取消收藏预设后刷新页面，确认状态符合预期；
   - 检查窄屏和宽屏布局，以及浏览器控制台是否有新增错误。

## 代码与内容约定

- 使用现有的原生 HTML、CSS 和 JavaScript 风格，避免无必要地引入框架或构建工具。
- 变量、函数和文件名保持语义清晰；新增用户可见文字时优先沿用现有的中英双语风格。
- 不要提交密码、服务器地址、备案信息、真实用户数据或其他敏感内容。部署变量仅放在本地的 `scripts/.env`。
- 文档中的功能描述必须能在当前代码中验证，避免承诺尚未实现的能力。

## 报告问题

使用 GitHub Issue 时，请尽量附上：复现步骤、预期与实际行为、浏览器/系统版本、截图或控制台报错。安全漏洞请按 [SECURITY.md](SECURITY.md) 中的方式私下报告。

## 行为准则

请在讨论中保持尊重、建设性和包容性。针对贡献本身提出具体、可执行的反馈。
