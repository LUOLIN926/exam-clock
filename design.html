颜色 / Colors

项目调色板灵感来自自然与大地，包含苔藓绿、赭石、陶土、奶油白等色调。所有颜色通过 CSS 变量统一管理，禁止在代码中使用硬编码色值。

变量名 色值 语义角色
--color-primary #4A6741 品牌主色，核心交互、激活状态
--color-primary-dark #3A5334 品牌深色，Hero 渐变、深色强调
--color-primary-light #6B8E63 品牌浅色，辅助背景、装饰
--color-secondary #A0522D 赭石色，活跃分节、次要强调
--color-accent #8B7355 强调色，文本高亮、分隔线
--color-earth #C4A77D 大地色，边框、分隔线
--color-cream #FAF8F5 奶油白，页面背景
--color-cream-dark #F5F2ED 深奶油，卡片/表单背景
--color-text #3D3229 主文字色，正文内容
--color-text-light #6B5D4D 浅色文字，辅助说明
--color-text-muted #9A8B78 禁用文字，标签、占位符
--color-success #5B8C5A 成功态，完成状态、确认
--color-warning #C9844A 警告态，当前分节、提醒
--color-danger #A65D4E 危险态，删除、错误
--color-sky #5B8A8A 装饰色，表单高亮、链接
--color-sky-dark #4A7575 深天空色，按钮深色变体
--color-autumn #9B8568 装饰色，编辑按钮
--color-autumn-dark #8B7355 深秋色，添加按钮

--color-lavender-dark #b283b6 深薰衣色，按钮深色变体
--color-moss #6B8E6B 装饰色，自然主题点缀
--color-rust #B8765A 装饰色，铁锈色点缀
使用指南
• 品牌色 — 用于核心操作按钮、激活状态、品牌元素，每屏仅一个 Primary 按钮
• 状态色 — 仅用于状态反馈（成功/警告/危险），不可用于装饰
• 装饰色 — 用于分节颜色编码、按钮功能区分、视觉点缀
• 大地色 — 统一用作边框和分隔线
• 奶油色 — 页面背景和卡片背景，保持温暖有机感
Do ✅ • 使用语义变量名（--color-danger）替代十六进制值
• 确保在白底上文字使用 Text / Text Light
• 状态色与对应组件语义一致（删除用 danger，完成用 success）
Don't ❌ • 使用装饰色（lavender, sky, moss）作为行动按钮背景
• 在同一视图中使用超过 2 种装饰色
• 在代码中硬编码颜色值（见下方注意事项）
// 正确使用 .section { border-left: 4px solid var(--color-primary); } .btn-delete { background: var(--color-danger); color: white; } // 错误 — 硬编码色值 .section { border-left: 4px solid #4A6741; } /* 应使用 --color-primary */

字体排印 / Typography

项目使用 Noto Serif SC 衬线字体栈，回退到 Noto Serif 和系统衬线字体，确保 100% 免费商用合规。

变量名 值 用途
--font-family 'Noto Serif SC', 'Noto Serif', serif 全局字体栈
--font-weight-regular 400 正文默认、描述文本
--font-weight-medium 500 输入值、时间显示、轻微强调
--font-weight-semibold 600 标题常用、卡片头部、分节名
--font-weight-bold 800 按钮、CTA、需要强调的数字
字号层级
• H1 (1.8em) — 页面主标题，每页仅一个
• H2 (1.5em) — 分节标题，如「颜色」「字体」「间距」
• H3 (1.3em) — 子分节标题，如「字重」「字号大小」
• 大正文 (1.1em) — 重要说明文本、提示信息
• 正文 (1em) — 默认正文大小
• 小正文 (0.9em) — 辅助说明、标签
• 微文本 (0.8em) — 状态标签、代码注释、辅助信息
Do ✅ • 使用 token 名称（--font-weight-semibold）替代原始数值
• 保持标题层级连续，不跳级（H1 → H2 → H3）
• 计时器显示使用 font-variant-numeric: tabular-nums 防止数字跳动
Don't ❌ • 正文段落使用 Bold 字重
• 在同一组件内混用 serif 和 sans-serif 字体
• 跳过标题层级（如 H1 直接使用 H3）
/* 标题层级 */ h1 { font-size: 1.8em; font-weight: 800; color: var(--color-primary-dark); } h2 { font-size: 1.5em; font-weight: 800; color: var(--color-primary-dark); } h3 { font-size: 1.3em; font-weight: 600; color: var(--color-text); } /* 计时器数字 — 等宽数字 */ .timer-display { font-variant-numeric: tabular-nums; font-weight: var(--font-weight-bold); }
font-family: 'Noto Serif SC', 'Noto Serif', serif

间距 / Spacing

统一的间距刻度系统，基于 4px 倍数递进。所有间距必须使用 Token，禁止使用中间值。

变量名 值 典型用途
--spacing-xs 4px 紧密内部间距（图标与文字、复选框与标签）
--spacing-sm 8px 组件内元素间距（按钮内边距、卡片内部间距）
--spacing-md 16px 组件内边距、表单字段间距
--spacing-lg 24px 组件组间距、分节内部间距
--spacing-xl 32px 大分节间距
--spacing-2xl 48px 页面级间距、Hero 区域内边距
Do ✅ • 不确定时选择更大一级的 Token，留出呼吸空间
• 同一组件内使用一致的间距（不要混用 sm 和 md）
• 表单字段用 md，组件组用 lg
Don't ❌ • 使用 Token 之间的自定义像素值（如 12px、20px）
• 内部组件使用 xl 或 2xl 作为内边距
• 在同一层级混用不同级别的间距
/* 卡片间距示例 */ .section-card { padding: var(--spacing-md); /* 16px 内边距 */ gap: var(--spacing-sm); /* 8px 内部间距 */ } /* 页面级容器 */ .design-container { padding: var(--spacing-xl) var(--spacing-md) var(--spacing-2xl); /* 32px top, 16px sides, 48px bottom */ }

圆角 / Border Radius

有机感的圆角风格，从紧凑到全圆。圆角大小应与元素尺寸成比例。

变量名 值 应用于
--radius-sm 8px 按钮、输入框、标识、小型交互元素
--radius-md 12px 卡片、弹窗、下拉菜单、中等容器
--radius-lg 16px 计时器显示、Hero 区域、大型展示区域
--radius-xl 24px Hero 横幅、页面头部、非常突出的容器
--radius-full 9999px 药丸形标识、CTA 按钮、进度条轨道
Do ✅ • 小型交互元素（按钮、输入框）使用 sm，呈现专业感
• 药丸形状态标识使用 full
• 计时器等大型展示元素使用 lg
Don't ❌ • 在小按钮上使用 xl 或 full（比例失调）
• 同一组件组内混用不同级别的圆角
• 对大容器使用 sm 圆角（看起来过于僵硬）
/* 按钮 — sm 圆角 */ .controls { border-radius: var(--radius-sm); } /* 卡片 — md 圆角 */ .section-card { border-radius: var(--radius-md); } /* CTA 按钮 — full 药丸形 */ .cta-button { border-radius: var(--radius-full); } /* 计时器 — lg 圆角 */ .timer-display { border-radius: var(--radius-lg); }

阴影 / Shadows

柔和的有机阴影，基于 rgba(61, 50, 41) 的棕色调。阴影用于创建层次感和深度。

变量名 值 高度层级
--shadow-sm 0 2px 8px rgba(61, 50, 41, 0.08) 静态卡片、统计卡片、微妙的表面深度
--shadow-md 0 4px 16px rgba(61, 50, 41, 0.12) Hover 状态卡片、下拉菜单、弹窗内容
--shadow-lg 0 8px 32px rgba(61, 50, 41, 0.16) Hero 区域、悬浮元素、弹窗、重要覆盖层
Do ✅ • Hover 时增加阴影以传达「浮起」效果（sm → md）
• 仅对视觉上「悬浮」在内容之上的元素使用 lg
• 静态卡片使用 sm 阴影
Don't ❌ • 对已有强边框的元素使用阴影（会过度拥挤）
• 在同一页面层级使用多种阴影级别
• 对背景元素或内联元素使用阴影
/* 卡片 — 静态 */ .card { box-shadow: var(--shadow-sm); } /* 卡片 — Hover 浮起 */ .card:hover { box-shadow: var(--shadow-md); transform: translateY(-2px); } /* Hero 区域 — 强悬浮 */ .hero { box-shadow: var(--shadow-lg); }

过渡 / Transitions

统一的动画时长和缓动曲线。将鼠标悬停在下方色块上查看效果。

变量名 值 用途
--transition-fast 0.15s ease-out 微交互（图标旋转、颜色闪变、切换状态）
--transition-normal 0.3s ease-out 标准 Hover 效果、按钮状态变化、卡片浮起
--transition-slow 0.5s ease-out 页面级过渡、分节展示动画、进度条填充
Do ✅ • 对频繁变化的属性（复选框、图标旋转）使用 fast
• 大多数 Hover / Focus 过渡使用 normal
• 仅对 transform、opacity、background-color 使用过渡
Don't ❌ • 按钮 Hover 使用 slow — 感觉迟钝不灵敏
• 对 width、height、margin 等影响布局的属性使用过渡
• 使用自定义缓动函数（统一用 ease-out）
/* 按钮 Hover */ .controls { transition: transform var(--transition-fast), background var(--transition-fast); } .controls:hover { transform: translateY(-2px); } /* 进度条填充 */ .progress-fill { transition: width var(--transition-slow); }

按钮 / Buttons

所有按钮使用 min-height: 40px 触摸目标，font-weight: bold，font-size: 14px。Hover 时 translateY(-2px) + 阴影增强。

组件结构
min-height: 40px · padding: 10px 16px · font-size: 14px · font-weight: bold · border-radius: var(--radius-sm)
变体 背景色 文字色 边框 用途
Primary --color-primary White None 主要操作（Start / Pause / Reset）
Functional 功能色变量 White None 保存、应用、返回、添加等功能操作
Danger --color-danger White None 仅用于破坏性操作（删除）
Secondary --color-cream-dark --color-text-light 1px --color-earth 次要操作、导航
CTA --color-primary White None 落地页主行动按钮，radius-full
Toggle White / 激活色 --color-text-light / White 动态 筛选、分类切换
Disabled --color-text-muted White None 暂时不可用状态，opacity: 0.6
Do ✅ • 每屏仅一个 Primary 按钮，确保视觉焦点
• 危险按钮仅用于破坏性操作（删除、移除）
• 暂时不可用时使用 Disabled 态
Don't ❌ • 在非破坏性操作中使用 Danger 按钮
• 同组按钮混用不同变体（暗示不同重要性）
• CTA 按钮用于表单内部（CTA 仅用于落地页）
/* 主要按钮 */ .btn-primary { background: var(--color-primary); color: white; border: none; padding: 10px 16px; font-size: 14px; font-weight: bold; min-height: 40px; border-radius: var(--radius-sm); cursor: pointer; transition: transform var(--transition-fast); } .btn-primary:hover { transform: translateY(-2px); } /* 禁用态 */ .btn-primary:disabled { cursor: not-allowed; opacity: 0.6; }

输入框 / Inputs

所有输入框使用 1px solid var(--color-earth) 边框，--radius-sm 圆角，白色背景（时间/日期显示使用 cream-dark 背景）。

组件结构
border: 1px solid var(--color-earth) · border-radius: var(--radius-sm) · padding: 8px 12px · font-size: 14px · font-family: var(--font-family)
变体 背景色 特点 用途
文本输入 White 标准边框 + 聚焦 sky 环 自由文本（名称、描述）
时间/日期显示 --color-cream-dark cream-dark 背景，可点击 自定义选择器触发器
选择器 White 自定义 div 下拉面板，.custom-select，arrow 旋转 180° 从已知集合中选择
Do ✅ • 所有输入框始终搭配可见 Label
• 聚焦时使用 sky 色边框提示（border-color: var(--color-sky)）
• 时间/日期显示保持 cream-dark 背景以区分于可编辑输入框
• 选择器下拉面板使用 sky 边框 + shadow-md
Don't ❌ • 同一类型的输入框混用背景色
• 移除输入框边框（降低可发现性）
• 时间/日期显示做成普通输入框（用户会误以为可编辑）
• 选择器下拉面板使用浏览器默认样式

计时器显示 / Timer Displays

各类计时器的视觉样式，带不同颜色的边框标识状态。所有计时器使用 tabular-nums 防止数字跳动。

显示类型 边框色 背景色 文字色 用途
Real-time --color-primary var(--color-section-completed-bg) --color-primary-dark 当前实时时钟时间
Countdown --color-secondary var(--color-section-active-bg) --color-secondary 考试倒计时
Section Timer --color-warning var(--color-timer-section-bg) --color-warning 当前分节倒计时
组件结构
border: 2px solid 状态色 · border-radius: var(--radius-lg) · padding: 15px · font-size: 1.5em · font-weight: bold · font-variant-numeric: tabular-nums
Do ✅ • 所有时间显示使用 tabular-nums（等宽数字）
• 边框色与背景色调保持一致（绿色配绿底，警告色配黄底）
• 计时器字号不低于 1.2em（确保可读性）
Don't ❌ • 将计时器组件用于非时间内容
• 改变计时器字号过小（低于 1.2em 会失去视觉识别性）
• 边框色与背景色不匹配（如红色边框配绿色背景）

进度条 / Progress Bar

使用渐变填充的进度条，展示考试完成进度。每屏仅一个进度条。

组件结构
轨道：background: var(--color-cream-dark) · border-radius: var(--radius-full) · height: 12px
填充：background: linear-gradient(var(--color-primary), var(--color-success)) · border-radius: var(--radius-full) · transition: width var(--transition-normal)
Do ✅ • 使用 transition: width var(--transition-normal) 平滑更新进度
• 进度填充使用 primary-to-success 渐变表示正向进度
• 每屏仅展示一个进度条（考试总进度）
Don't ❌ • 用进度条展示非百分比数据
• 同时展示多个进度条
• 使用非渐变的纯色填充（缺乏层次感）
/* 进度条轨道 */ .progress-bar { height: 12px; background: var(--color-cream-dark); border-radius: var(--radius-full); overflow: hidden; } /* 进度条填充 */ .progress-fill { height: 100%; background: linear-gradient(90deg, var(--color-primary), var(--color-success)); border-radius: var(--radius-full); transition: width var(--transition-normal); }

导航与结构 / Navigation & Structure

页面底部导航栏和内联设置控件的样式规范。

元素 背景色 边框 文字色 用途
底部栏 --color-cream-dark 顶部 1px --color-earth --color-text-muted 备案信息、次要链接
内联设置控件 --color-cream-dark 1px --color-earth --color-text-light 显示设置切换（实时、倒计时、分节计时）
Do ✅ • 底部栏内容保持精简（最多 1-2 个链接）
• 底部栏链接使用 earth 色边框保持视觉一致
• 内联设置控件不超过 3 个复选框/按钮组
Don't ❌ • 在底部栏放置主要导航
• 在一组内联设置控件中放置过多选项
• 底部栏链接使用高亮颜色（应保持低调）
/* 底部栏 */ .bottom-bar { background: var(--color-cream-dark); padding: 15px; border-top: 1px solid var(--color-earth); text-align: center; font-size: 0.85em; color: var(--color-text-muted); } .bottom-bar a { color: var(--color-primary); } .bottom-bar a:hover { text-decoration: underline; } /* 内联设置控件 */ .inline-display-settings { display: flex; flex-wrap: wrap; gap: 10px; align-items: center; background: var(--color-cream-dark); border: 1px solid var(--color-earth); border-radius: var(--radius-sm); padding: 10px; }
